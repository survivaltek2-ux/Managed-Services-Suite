#### Authorizations

[​](https://docs.letshum.com/api-reference/service-availability/get-internet-service-availability#authorization-authorization)

Authorization

string

header

required

Bearer token authentication using API tokens.
Include the token in the Authorization header as: `Authorization: Bearer <token>`

Obtain tokens through your Hum API account dashboard or contact [support@letshum.com](mailto:support@letshum.com) for access.

#### Path Parameters

[​](https://docs.letshum.com/api-reference/service-availability/get-internet-service-availability#parameter-token)

token

string

required

The session token identifying the specific session
The session token provided by the Hum API. Used to connect the response to the session in the client system.

Example:

`"XqCmeTVgYXrbWrZFZEymkD"`

#### Response

200

application/json

Internet service providers, available plans, and logos/urls for a given service address.

[​](https://docs.letshum.com/api-reference/service-availability/get-internet-service-availability#response-message)

message

string

required

A message returned by the API. Includes a human-readable message about the status of the request.

Example:

`"What happened in the most recent request."`

[​](https://docs.letshum.com/api-reference/service-availability/get-internet-service-availability#response-request-status)

request\_status

enum<string>

required

A status code summarizing the outcome of this response.

- ok: Request was successful
- warning: Request was processed but with some issues
- error: Request failed

Available options:

`ok`,

`warning`,

`error`

Example:

`"ok"`

[​](https://docs.letshum.com/api-reference/service-availability/get-internet-service-availability#response-data)

data

object

required

[​](https://docs.letshum.com/api-reference/service-availability/get-internet-service-availability#response-meta)

meta

object

required

Metadata about the session. Can be used to track the status of the session, time since creation, and time since last update.
